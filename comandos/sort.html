SORT

El comando SORT sirve para ordenar el texto de un archivo.

Parámetros:

-b, --ignore-leading-blanks

Ignora espacios en blanco

-d, --dictionary-order

Considera solo espacios en blanco y caracteres alfanuméricos

-f, --ignore-case

Convierte caracteres en minúscula a mayúscula para comparar

-g, --general-numeric-sort

Compara según el valor numérico general

-i, --ignore-nonprinting

Considera solo caracteres imprimibles

-M, --month-sort

Compara (desconocido) < 'ENE' < ... < 'DIC'

-h, --human-numeric-sort

Compara números legibles para humanos (por ejemplo, 2K 1G)

-n, --numeric-sort

Compara según el valor numérico de la cadena

-R, --random-sort

Baraja, pero agrupa claves idénticas. Consulte shuf(1)

--random-source=FILE

Obtiene bytes aleatorios de FILE

-r, --reverse

Invierte el resultado de las comparaciones

--sort=WORD

Ordena según WORD: numérico general -g, numérico humano -h, mes -M, numérico -n, aleatorio -R, versión -V

-V, --version-sort

Ordenación natural de números (de versión) dentro del texto

Otros parametros:

--batch-size=NMERGE

Fusiona como máximo NMERGE entradas a la vez; para más, use archivos temporales

-c, --check, --check=diagnose-first

Verifica la entrada ordenada; no ordena

-C, --check=quiet, --check=silent

Como -c, pero no informa sobre la primera línea incorrecta

--compress-program=PROG

Comprime temporales con PROG; descomprímelos con PROG -d

--debug

Anota la parte de la línea utilizada para ordenar y advierte sobre el uso cuestionable a stderr

--files0-from=F

Lee la entrada de los archivos especificados por nombres terminados en NUL en el archivo F; Si F es -, lea los nombres desde la entrada estándar

-k, --key=KEYDEF

Ordena mediante una clave; KEYDEF da ubicación y tipo

-m, --merge

Fusiona archivos ya ordenados; no ordena

-o, --output=FILE

Escribe el resultado en FILE en lugar de la salida estándar

-s, --stable

Estabiliza la ordenación deshabilitando la comparación de último recurso

-S, --buffer-size=SIZE

Utiliza SIZE para el búfer de memoria principal

-t, --field-separator=SEP

Utiliza SEP en lugar de la transición de no blanco a blanco

-T, --temporary-directory=DIR

Utiliza DIR para temporales, no $TMPDIR o /tmp; varias opciones especifican varios directorios

--parallel=N

Cambia el número de ordenaciones ejecutadas concurrentemente a N

-u, --unique

Con -c, verifica el orden estricto; sin -c, muestra solo el primero de una ejecución igual

-z, --zero-terminated

El delimitador de línea es NUL, no nueva línea

--help

Muestra esta ayuda y sale

--version

Muestra información de versión y sale

Ejemplos:

Mostrar archivo ordenado alfabeticamente (a-z).

sort myfile.txt

-!?

012345

abcdefg

cdefg

uvwxyz

xyz

Mostrar archivo ordenado alfabeticamente en reversa (z-a).

sort myfile.txt -r

xyz

uvwxyz

cdefg

abcdefg

012345

-!?
